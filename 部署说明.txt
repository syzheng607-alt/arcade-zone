================================================

📌 关于 DEPLOYMENT_NOT_FOUND 错误
----------------------------------
这个错误表示：
✗ 你访问的部署 URL 不存在
✗ 部署可能被删除了
✗ 使用了错误的部署 ID 或 URL
✗ 没有访问权限

🎯 解决方案：确保使用正确的、最新的部署 URL


方法一：使用 Vercel 网页界面（最简单）⭐
=====================================

1. 访问 https://vercel.com 并登录

2. 点击 "Add New..." → "Project"

3. 如果你有 GitHub 账户：
   a. 在 GitHub 上创建一个新仓库
   b. 上传这个文件夹的所有文件到 GitHub
   c. 在 Vercel 中选择这个 GitHub 仓库
   d. 点击 "Deploy"

4. 如果没有 GitHub：
   a. 下载并安装 GitHub Desktop: https://desktop.github.com
   b. 用 GitHub Desktop 创建仓库并推送代码
   c. 然后在 Vercel 中导入

5. 部署完成后，Vercel 会给你两个 URL：
   📍 生产环境: https://your-project.vercel.app
   📍 预览环境: https://your-project-git-main-username.vercel.app


方法二：使用 Vercel CLI（需要先安装 Node.js）
==========================================

1. 安装 Homebrew（如果还没有）：
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

2. 安装 Node.js：
   brew install node

3. 安装 Vercel CLI：
   npm install -g vercel

4. 同意 Xcode 许可（如果需要）：
   sudo xcodebuild -license

5. 在终端中进入项目目录：
   cd "/Users/littlerock/Desktop/OKComputer_Arcade Game Site Design"

6. 登录 Vercel：
   vercel login

7. 部署项目：
   vercel

8. 按提示操作：
   - Set up and deploy? → Yes
   - Which scope? → 选择你的账户
   - Link to existing project? → No
   - Project name? → arcade-zone
   - In which directory? → ./
   - Override settings? → No

9. 部署到生产环境：
   vercel --prod


📊 避免 DEPLOYMENT_NOT_FOUND 的最佳实践
======================================

✅ DO（应该做的）：
- 使用生产环境 URL (*.vercel.app)
- 配置自定义域名获得稳定 URL
- 使用环境变量而不是硬编码 URL
- 在 Vercel 控制台检查部署状态
- 保存正确的部署 URL

❌ DON'T（不应该做的）：
- 不要硬编码特定的部署 ID
- 不要假设预览部署永久存在
- 不要在没有错误处理的情况下访问部署
- 不要缓存旧的部署 URL
- 不要在被删除后还尝试访问旧部署


🔍 常见问题排查
==============

问题1: "DEPLOYMENT_NOT_FOUND"
解决：
- 在 Vercel 控制台查看部署历史
- 确认使用的是正确的生产 URL
- 重新部署项目：vercel --prod

问题2: "资源加载失败"
解决：
- 检查 resources/ 文件夹是否完整
- 确保所有图片都已上传
- 查看浏览器控制台的具体错误

问题3: "页面显示 404"
解决：
- 确认 index.html 在项目根目录
- 检查 vercel.json 配置是否正确
- 在 Vercel 控制台查看构建日志
